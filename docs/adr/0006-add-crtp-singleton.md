# ADR-006: CRTP单例模式

用CRTP写单例模式来减少代码膨胀， 目前只有日志 log 使用的单例模式。

-- 2026-06-23 Arsenova
## 决定
本项目对于CRTP单例带来的模版膨胀代价可以忽略不计， 本身可能就几个单例类。
使用CRTP减少单例模版代码的编写。

## 代价
给未来的自己或者其他人，不要迷信CRTP模版。
我必须吐槽CRTP模版，哪怕我在写数据结构都几乎写的base hook CRTP。
- 侵入式太强，破坏耦合，基类想直接调用子类私有构造要么公开接口要么声明友元类。
- 代码膨胀， 这个项目大概不会存在，本身就是每个单例类会各自生成一份独立的积累代码， 会生成大量的机器码，可执行文件体积变大。

解决方案， 如果有以上代价，老老实实就在单例类写懒汉风格的 Magic Statis（局部静态变量线程安全)。

## 最后
我了解到一种在C++中实现Java Spring这种在运行时依赖注入容器的方案， 或许可以编写一个这种容器来处理单例模式。 但是本项目完全没有必要。
